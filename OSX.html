AFS to OS X

Tutorial:

  1. Download the application here.


  2. Unzip the file.


  3. Double click the application. Even if the application is not in the downloads folder, it will still download your files to the Downloads folder.


  4. Certify that you want to run the software.


  5. A terminal window will appear, enter your uniqname as prompted.


  6. You will be asked to verify that you are connecting to a University server if you have not connected to his server before from this computer. Type the word “yes” where prompted.


  7. Enter your password.


  8. All non-hidden files that you have access to in your AFS directory will be copied to your computer.


  9. Your files will appear in a directory with the name matching your uniqname, and this directory will always be in your downloads folder. It will be an exact replica of your AFS directory, minus hidden files.


This tool consists of two scripts, an AppleScript and a Bash script. The Applescript collects the filepath to the current directory, opens a terminal window, and executes the Bash script. The Bash script collects the user’s uniqname, and uses it to open an ssh connection with rsync in order to gain access to the user’s AFS space. The user then must authenticate and acknowledge they are connecting to a University of Michigan server, and then rsync utility will copy the user’s entire AFS space to a directory named their uniqname in their downloads directory. The download can be interrupted at anytime without issue. Simply re-run the application and the download will pick up where it was interrupted.

Notes for power users: this rsync is run with -p enabled, so that permissions will be preserved. Hidden files are excluded from being copied.
